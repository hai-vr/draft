If you use UdonSharp, you already know how to use Editor Scripting

If you use UdonSharp, you don't need to commit to learn Editor Scripting.

It's likely that you already know how to use Editor Scripting. Being able to use Editor Scripting at the right time can give you a productivity boost. Learn in small chunks.

To get started, it takes a couple minutes:

  • Create a GameObject, click Add Component > New script, name it MyStuff, and click Create.
  • Replace the contents of the script with something like:
using UnityEngine;

public class MyStuff : MonoBehaviour
{
    public float myValue;
}
  • Use the Project Explorer to create a folder called Editor/ somewhere. The name of the folder must be exactly Editor. It can be located in any folder.
  • Create a new C# Script asset within that folder and name that script MyStuffEditor
  • Replace its contents with the following:
using UnityEditor;
using UnityEngine;

[CustomEditor(typeof(MyStuff))]
public class MyStuffEditor : Editor
{
    public override void OnInspectorGUI()
    {
        DrawDefaultInspector();
        if (GUILayout.Button("Run"))
        {
            Run();
        }
    }

    private void Run()
    {
        var myValue = ((MyStuff)target).myValue;
        Debug.Log($"The value is {myValue}");
    }
}

Your inspector now has a button labelled Run that you can click, and it will print the value of a field from that object.

There is much more than that possible but this simple script should already get you places.

Think of some of the UdonBehaviours that you run in-game, that you could run one time within the Editor instead.

Placing the Editor script a folder called exactly Editor/ is important because it will prevent the build from crashing, as this depends on UnityEditor.

Detect when a VRChat build starts

It is possible to detect when a VRChat build is going to start, using the IVRCSDKBuildRequestedCallback callback interface.

Interrupting the build from starting is possible by returning false.

Detect entering Play mode

It is possible to detect when Play mode is about to be entered, using the EditorApplication.playModeStateChanged callback.

Preventing from entering Play mode is possible by detecting that the state is equal to PlayModeStateChange.ExitingEditMode and then setting EditorApplication.isPlaying to false.

Consider using Handles instead of Gizmos

Handles may have some functionality that Gizmos do not.

For instance, Handles.DrawLine has more options related to the rendering lines passing through geometry than Gizmos.DrawLine.

Draw attention to an object in the hierarchy or the project

To ping an asset, use EditorGUIUtility.PingObject(whichObject).

When editing an VRCExpressionsMenu asset, use AssetDatabase.ForceReserializeAssets() to save it

When using Editor Scripting to edit an existing VRCExpressionMenu asset, use the following to save it:

void SaveExistingExpressionMenu(VRCExpressionMenu assetToSave)
{
    var path = AssetDatabase.GetAssetPath(assetToSave);
    AssetDatabase.ForceReserializeAssets(new []{ path });
}
